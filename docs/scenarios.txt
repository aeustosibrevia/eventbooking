Дані в системі зберігаються в базі даних SQLite у вигляді таблиць для сутностей User, Event та Booking, керування якими здійснюється відповідними сервісами NestJS.

Під час створення користувача формується новий запис у таблиці User з унікальним ідентифікатором, що розширює набір даних у базі.

При створенні події аналогічно формується новий запис у таблиці Event з унікальним ідентифікатором і зберігається у базі даних.

Створення бронювання є ключовим сценарієм агрегації даних: перед додаванням запису в таблицю Booking перевіряється існування користувача та події. Новий запис Booking зв’язує сутності User та Event через зовнішні ключі userId та eventId.

Отримання бронювань не змінює стан даних, а виконує їх агрегацію шляхом фільтрації записів у таблиці Booking за eventId або повернення повного списку.

Операції оновлення та видалення сутностей у системі не реалізовані, тому всі зміни даних мають адитивний характер.
